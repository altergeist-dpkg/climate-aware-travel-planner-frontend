# Specification: Docker Configuration for Next.js

## Overview
This specification defines the production-ready Docker setup for the Next.js frontend application. It uses a multi-stage build pattern to minimize the final image size and securely injects the `NEXT_PUBLIC_API_URL` environment variable during the build process.

## 1. Next.js Configuration (`next.config.mjs` or `next.config.js`)
To optimize the Docker image, Next.js needs to be configured to build as a standalone application.
* **Update the config:** Add `output: 'standalone'` to the `nextConfig` object. This tells Next.js to trace and copy only the necessary files for production into a `.next/standalone` directory.

## 2. Dockerfile (`Dockerfile`)
Create a `Dockerfile` in the root of the project using the official Node Alpine image (e.g., `node:20-alpine`).
* **Stage 1: Dependencies (`deps`)**
  * Copy `package.json` and `package-lock.json` (or `npm-shrinkwrap.json` / `yarn.lock`).
  * Run `npm ci` (or the equivalent strict install command).
* **Stage 2: Builder (`builder`)**
  * Copy the `node_modules` from the `deps` stage.
  * Copy the rest of the application code.
  * **Environment Variable Injection:**
    * Define `ARG NEXT_PUBLIC_API_URL`.
    * Define `ENV NEXT_PUBLIC_API_URL=$NEXT_PUBLIC_API_URL`. This is critical because Next.js bakes `NEXT_PUBLIC_` variables into the client bundle at build time.
  * Run `npm run build`.
* **Stage 3: Runner (`runner`)**
  * Set `NODE_ENV=production`.
  * Copy the generated `.next/standalone` directory and `.next/static` folder from the `builder` stage.
  * Expose port 3000.
  * Command to run: `node server.js` (which is generated by the standalone output).

## 3. Docker Ignore (`.dockerignore`)
Create a `.dockerignore` file to prevent unnecessary files from being copied into the Docker context, speeding up the build.
* Exclude: `node_modules`, `.next`, `.git`, `coverage`, `specs`, and markdown files.

## Execution Instructions for Cursor
1. Read the specification.
2. Update `next.config.mjs` to include the standalone output.
3. Create the `.dockerignore` file.
4. Create the multi-stage `Dockerfile` strictly implementing the `ARG` and `ENV` mapping for `NEXT_PUBLIC_API_URL` in the build stage.
